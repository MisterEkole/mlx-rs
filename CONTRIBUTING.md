# Contributing to mlx-rs

Thank you for contributing! Please follow these guidelines.

## Quick Setup

1. **Prerequisites**: Rust, Xcode CLI tools, CMake
2. **Build mlx-c**:
   ```bash
   git clone https://github.com/ml-explore/mlx-c.git
   cd mlx-c && mkdir build && cd build
   cmake .. -DCMAKE_BUILD_TYPE=Release && make -j
   ```

3. **Setup mlx-rs**:
   ```bash
   export MLX_C_PATH=/path/to/mlx-c
   git clone https://github.com/MisterEkole/mlx-rs.git
   cd mlx-rs
   ./check_env.sh
   cargo build -p mlx-sys
   cargo build
   ```

## How to Contribute

**Find something to work on:**
- Check [API_COVERAGE.md](API_COVERAGE.md) for unimplemented features
- Look at open issues labeled `good first issue` or `help wanted`
- Priority areas: FFT, optimizers, quantization, file I/O

**Implementation steps:**
1. Add FFI bindings in `mlx-sys`-- nindings auto generated by rust bindgen in most cass
2. Create safe wrapper in `mlx` crate
3. Write tests
4. Add documentation
5. Update [API_COVERAGE.md](API_COVERAGE.md)

## Pull Request Guidelines

1. **Create a feature branch**: `git checkout -b feature/your-feature`
2. **Write clear commits**: Use present tense, reference issues
3. **Add tests**: Required for all new functionality
4. **Update documentation**: Add rustdoc comments, update API_COVERAGE.md if needed
5. **Submit PR** with:
   - Clear description of changes
   - Reference to related issues
   - Examples (if applicable)
